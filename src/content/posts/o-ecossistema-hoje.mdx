O post anterior sobre o ecossistema (30 de junho) mostrava 3 projetos começando a se estruturar. Duas semanas depois, o cenário é outro:

| Projeto | Junho | Hoje | + |

O ecossistema deixou de ser um conjunto de scripts e virou uma plataforma operacional.

## 🌐 O mapa do ecossistema

Cada projeto tem um papel específico e se conecta aos outros por canais bem definidos:

```
                    ┌─────────────┐
                    │   Hermes    │ ← Cérebro que orquestra TUDO
                    │  (Agente)   │
                    └──────┬──────┘
          ┌────────────────┼────────────────┐
          ▼                ▼                ▼
   ┌──────────┐    ┌──────────┐    ┌──────────────┐
   │  Arachne │    │ Capivara │    │   Dogwalk    │
   │  :9000   │◄──►│  :8001   │    │  seu.pet     │
   │ MCP+RAG  │    │ Hub+Admin│    │ Stripe+GPS   │
   └──────────┘    └──────────┘    └──────────────┘
          │              │
          ▼              ▼
   ┌──────────┐    ┌──────────────┐
   │ LifeLog  │    │  Portifolio  │
   │ Blog 26  │    │ Next.js 16   │
   │ posts    │    │ 5 mini-games │
   └──────────┘    └──────────────┘
          │
          ▼
   ┌──────────────┐
   │  TatuEngine  │ ← O mais novo, o mais ambicioso
   │ C++23 + CUDA │
   └──────────────┘
```

**Conexões reais hoje:**
- **Capivara → Arachne**: Health check e sincronização de dados do admin
- **Capivara → Dogwalk**: Dashboard de revenue, leaderboard, stats
- **Capivara → Portifolio**: Contact form + CV downloads via API
- **LifeLog → Todos**: O blog que conta a história de cada projeto
- **Arachne MCP → Todos**: Ferramentas expostas pra agentes IA
- **Dogwalk API → Website**: Cloudflare Pages + Worker com HMAC Stripe

Não é uma arquitetura planejada no papel — cresceu organicamente, e isso tem vantagens e dívidas.

## 🦔 5. TatuEngine — o mais ambicioso

**289 commits · 400+ testes · C++23 + CUDA · SSM × Phase Field**

O projeto mais ousado: uma engine de inferência que mistura State Space Models com teoria de campo de fase. Roda na GPU (252× speedup comprovado) com compressão block-encoded de 114×.

```
TatuEngine
 ├── BlockCodec → 32×32 blocos ternários, 27 GB → 245 MB
 ├── GPU kernel → 32 threads/CTA, warp shuffle, 252× speedup
 ├── GenerationContext → Agente autopoiético com LTM, ToolUse, GoalStack
 ├── PhaseField → Campo potencial O(1), decisões SPLIT/PRUNE/EXPLORE
 ├── CLI tatu → generate, mcp, tool, info
 ├── Sandbox Híbrido → 51/51 testes, path validation
 └── MCP Server → FastMCP, 4 tools (stdio/SSE)
```

BitMamba 1B já treinado e validado. O pipeline de treino SFT com Adafactor resolveu 4 fontes de NaN que travavam o treinamento. O post *SFT do TatuEngine* documenta a saga.

## 🧠 Aprendizados

**1. Ecossistema não se planeja — se cultiva.**

Nenhum desses projetos nasceu de um roadmap. O Arachne era um scraper, o Capivara um painel simples, a TatuEngine uma curiosidade acadêmica. O que os uniu foi a necessidade real de cada momento.

**2. Um cérebro central vale mais que mil scripts.**

Antes do Hermes, cada projeto tinha seus próprios scripts de backup, health check e deploy. Hoje é um agente só que conhece todos eles. A sinergia entre projetos veio do orquestrador, não da arquitetura.

**3. Testes são a cola que mantém tudo junto.**

2.818 testes no Arachne. 219 no Portifolio. 400+ na TatuEngine. E toda vez que quebro algo, são eles que me avisam antes do deploy. Teste não é custo — é segurança pra poder mexer em produção sem medo.

**4. Stacks diferentes convivem quando a interface é limpa.**

FastAPI, Next.js, Astro, C++23 — cada projeto escolheu a ferramenta certa pro problema. A comunicação entre eles é via API REST + MCP. Não preciso de um monolito se tenho contratos claros.

**5. O que não está documentado não existe.**

O LifeLog nasceu dessa necessidade. Não adianta construir 5 projetos se ninguém (inclusive eu) sabe como eles funcionam, por que foram construídos, e o que aprenderam no caminho.
