## Introdução

No projeto Arachne, estamos sempre metendo a mão nos pipelines RAG (Retrieval-Augmented Generation) para garantir que o sistema seja rápido como uma flecha e preciso como um sniper.

Neste post, vou te contar um pouco sobre como o FTS5 do SQLite e as buscas vetoriais estão trabalhando juntas.

### Full-Text Search com FTS5

No mundo da busca, a velocidade é tudo. Aí que entra o SQLite FTS5, uma ferramenta poderosa para fazer buscas rápidas em textos grandes.

### Busca Vetorial

E não podemos esquecer das buscas vetoriais! São essenciais para lidar com consultas mais complexas e semânticas.

## Configurando o SQLite FTS5

```sql
CREATE VIRTUAL TABLE documents USING fts5(title, content);
```

```sql
INSERT INTO documents (title, content) VALUES ('Título do Documento', 'Conteúdo aqui...');
```

### Integração com Busca Vetorial

No Arachne usamos sqlite-vec pra busca vetorial direto no SQLite — sem depender de Faiss ou serviços externos.

> **Insight:** A combinação do FTS5 com sqlite-vec permite busca híbrida no mesmo banco. O melhor dos dois mundos sem sair do SQLite.

## Conclusão

A combinação do SQLite FTS5 e sqlite-vec no pipeline RAG do Arachne tem dado resultados incríveis em performance e precisão. O mais legal é que tudo roda local, sem depender de API externa.
